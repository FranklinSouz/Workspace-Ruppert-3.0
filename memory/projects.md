### Construção Técnica do Produto (definido 2026-03-28)

**Diferencial real:**
Não é chatbot. É "funcionário digital que Franklin instala, treina e gerencia". Três pilares: agente com memória/contexto + Franklin como gestor contínuo + expansão modular por área.

**Estrutura de workspace por cliente:**
```
workspace/
├── SOUL.md       → persona do agente (nome, tom)
├── BUSINESS.md   → contexto do negócio (serviços, horários, preços, políticas)
├── FAQ.md        → perguntas e respostas reais
├── RULES.md      → o que pode/não pode fazer
└── memory/
    ├── customers.md   → histórico de clientes recorrentes
    └── decisions.md   → decisões do dono sobre o agente
```

**Segurança:** cada cliente = workspace isolado, credenciais só na VPS de Franklin, contrato com cláusula LGPD.

**Templates por segmento:** template-clinica, template-imobiliaria, template-escola-idiomas (monta uma vez, replica sempre)

**Ordem de instalação:**
1. Onboarding (entrevista 1h — coleta serviços, horários, FAQ, tom, integrações)
2. Criar workspace + copiar template do segmento
3. Personalizar SOUL.md + BUSINESS.md + FAQ.md
4. Conectar WhatsApp
5. Conectar integrações (Google Agenda, CRM se Pro+)
6. Testar 20 conversas reais
7. Go live + monitorar 1ª semana
8. Handoff: Loom de 10min pro dono
9. Revisão mensal

**WhatsApp — estratégia por plano:**
- Starter: Zapi/EvolutionAPI (unofficial) + número dedicado separado. Custo ~R$150/mês absorvido na mensalidade. Risco baixo com volume baixo.
- Pro/Empresa: Meta Cloud API oficial. 1.000 conv/mês grátis (maioria dos clientes não passa). Acima disso ~R$0,30/conversa. Sem risco de ban.
- Medo de R$1k-3k/mês só acontece com 5.000+ conversas/mês — irrelevante pra pequenas empresas.
- Regra: Starter = Zapi + número separado. Pro pra cima = Meta oficial.
